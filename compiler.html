Lexical Analysis

Defined by the Regular Grammar. The Lexical Analyzer(or known as Scanner) do the analysis itself. The analysis consists of checking each character if it exists on alphabet and if it does not, then put this new one there (Maybe it sends a warning). As the check is going on, the Analyzer mount tokens on a symbols table. These tokens are mounted according to the analyzer see specific symbols.

Example: int foo;

The analyzer will do:

(I) ok, it exists. (N) ok, it exists. (T) ok, it exists. ( ) a space, so lets put these letters as a token on the symbols table.

(F) ok, it exists. (O) ok, it exists. (O) ok, it exists. (;) ok, it exists but theses letters will be a token and (;) another one.

Remember: There's no equal tokens on the same symbols table.

Token types:

  • Reserved Words
  • Comments
  • Literal/String/Constant
  • Identifiers(variables, non-terminals)
  • Symbols(Operators, termination)
