Concurrent and distributed computing

Introduction

First of all, we need to undestand what does concurrent and distributed means, right?!

Several of authors explain what does it means, but each one at his respective time and undestanding. We gonna adopt the following definitions:

Concurrent is a simultaneous execution of a program or system by several process that cooperate with each other.

Distributed System is a system consisting of multiple processors conected by a comunication network, and they comunicate with each other through messages sended over this network

In a project, the following goals needs to be considered(Even in a parallel system):

  • Access transparency -> How the data gonna be represented to the user
  • Location transparency -> The user doesn't know where resources are.
  • Replication transparency -> It has to have a copy to performance and possible failures
  • Persistence transparency -> The logic of transation
